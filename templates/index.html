01 / Flusso

Come ottenere una telemetria

Il catalogo guida il client dai dati generali fino al report, evitando richieste con parametri non validi.

  1. 01

    Scegli la stagione

    Recupera gli anni disponibili oppure usa direttamente una stagione supportata.

    GET /legacy/catalog/years
  2. 02

    Trova GP e sessione

    Il calendario restituisce eventi, località e codici sessione utilizzabili.

    GET /legacy/catalog/events?year=2024
  3. 03

    Seleziona i piloti

    Carica i piloti e verifica per ciascuno la disponibilità della telemetria.

    GET /legacy/catalog/drivers
  4. 04

    Scarica il PDF

    Richiedi il report singolo o il confronto tra due piloti.

    GET /get-telemetry

02 / Cache

Prima il file, poi FastF1

Ogni richiesta usa un nome deterministico. Se il PDF esiste, viene inviato immediatamente senza rigenerare la telemetria.

Richiesta Year + GP + Session + Driver
→
Cache lookup PDF disponibile?
Cache hit Invio diretto Nessun utilizzo FastF1
Cache miss Generazione PDF Massimo {{ max_concurrency }} generazioni simultanee

Politica LRU. I file usati meno recentemente vengono eliminati quando uno dei due limiti viene superato.

Pubblicazione atomica. Un PDF diventa disponibile soltanto dopo che la generazione è terminata correttamente.

Persistenza. La directory deve essere montata su un volume persistente per conservare i file tra deploy e riavvii.

03 / Utilizzo

Esempi di richiesta

La pagina non esegue chiamate. Gli esempi devono essere utilizzati da un client autorizzato sostituendo la chiave e i parametri.

Telemetria singola application/pdf 
curl \
  -H "{{ api_key_header }}: <API_KEY>" \
  "https://<HOST>/get-telemetry?year=2024&trackName=Monaco&session=Q&driverName=VER" \
  -o telemetry.pdf
Confronto piloti application/pdf 
curl \
  -H "{{ api_key_header }}: <API_KEY>" \
  "https://<HOST>/get-telemetry-compare?year=2024&trackName=Monaco&session=Q&driverA=VER&driverB=LEC" \
  -o comparison.pdf
